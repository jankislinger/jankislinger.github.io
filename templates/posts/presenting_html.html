Converting Tables to HTML: All the Ways

According to the Zen of Python, there should be one—and preferably only one—obvious way to do it. But when it comes to converting tables to HTML for presentation, that’s far from true. In this blog post we’ll explore all the options and help you decide which one is right for you.

Why Are There So Many Options?

Because Python has a rich ecosystem for processing tabular data—ranging from DataFrame libraries like Pandas or Polars to ORMs like SQLAlchemy. And on the presentation side, you have everything from Jupyter notebooks to web frameworks like Django or Flask.

To keep things focused, we’ll narrow our scope to:

  • DataFrames in Polars (with occasional detours to Pandas)
  • Presentation in Jupyter and Flask

Selection Criteria

As with any analysis, your choice depends on:

  • Audience: Who will consume the data?
  • Delivery: HTML may not always be the best format—sometimes PDF is more appropriate.
  • Existing Tooling: What libraries and frameworks are already part of your project?
  • Customization: How much raw HTML/CSS are you willing to write?

Start Simple

The most versatile approach is manual HTML templating in Flask. Let’s use the gapminder dataset as an example:

{{ table_01 | pyghl('html+jinja') | safe }}

And the Flask view:

import polars as pl
from flask import render_template

data = pl.DataFrame([
    ("Austria", 65, 12345),
    ("Belgium", 71, 23456),
], schema=["country", "life_exp", "gdp"] )
max_gdp = data.select(pl.col("gdp").max()).item()
return render_template(
    "table_template.html",
    table=data.to_dicts(),
    max_gdp=max_gdp
)
